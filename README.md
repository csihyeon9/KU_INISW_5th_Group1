**GNN의 분류 문제는 크게 3가지로 나뉜다 (Node, Edge, Graph)**
1. Node Classification: 노드의 클래스를 예측
2. Link Prediction: 노드 사이트 연결 여부를 예측 *
3. Graph Classification: 그래프의 클래스를 예측

**문제점**
1. 데이터를 어떻게 변환할 것인가?
GNN 을 사용하려면 tarbular data 를 Heterogeneous 한 그래프로 변환하는 작업이 필요하며, 
이를 위해 여러 툴들을 사용하게 된다

2. 새로운 데이터를 어떻게 에측할 것인가?
GraphSAGE: 실제로는 Inductive(매번 새로운 그래프로 학습) 한 워크로드가 많고(사기/이상 탐지 등), 
Heterogeneous graph 를 다루는 것이 일반적이다.
Graph sampling 은 학습에 사용한 그래프에서 새로운 노드(예측 대상이 되는) 의 이웃에 해당하는 노드들과 엣지를 뽑아내서 새로운 그래프를 만드는 것이다.

3. Framework : 보통 PyG(Pytorch Geometric)을 사용한다. 접근 용이성, Pytorch의 문법을 그대로 사용 가능

GNN을 이용하여 문제와 답 생성 및 문제 추천(Inductive) AI를 제작해보자 한다.
학습된 데이터를 기반으로 확인된 단어나 새로 들어온 단어를 노드로 지정
그리고 노드간의 관련성인 edge를 생성해야 한다. 이 방법으로는 아래 4가지를 기준으로 edge내용을 생성한다
edge 관계 유형에 사용할 관계 예시:
# 포함 관계 (Inclusion): 하나의 단어가 다른 단어를 포함하거나 소속된 관계.
# 종속 관계 (Dependency): 하나의 단어가 다른 단어에 의존하는 관계.
# 인과 관계 (Causal): 한 단어가 다른 단어의 원인이나 결과로 작용하는 관계.
# 상관 관계 (Correlation): 두 단어가 서로 상관관계를 가지는 경우.
# 구조적 관계 (Structural): 두 단어가 어떤 구조 내에서 관계를 맺고 있는 경우.
# 기능적 관계 (Functional): 특정 기능을 수행하기 위해 두 단어가 연결된 경우.
# 맥락적 관계 (Contextual): 특정 문맥에서 두 단어가 함께 사용되는 관계.

그러면 edge의 내용을 기반으로 문제와 답을 생성하는 방법을 사용하거나 추천_문제.db에서 사용자의 실력에 따라 문제를 추천하게 한다. 사용자의 학습에 가장 도움이 될거라 예측되는 문제를 추천하게 끔 하는 AI를 만들고자 한다. 

GCN (안된다고 했는데 나는 이걸 해볼거임 시도)
실사용 과정:
1. 웹 사이트 (기사)를 크롤링하여 키워드를 추출한다(Bert중 가장 효과적인 모델 사용). 추출된 단어를 저장.
2. 추출된 단어를 기존에 학습되었던 데이터를 기반으로 edge를 생성하거나 새롭게 생성한다.(시각화 제공)
3. edge의 관계 유형에 따라 문제를 생성 or 추천_문제.db 에서 가져온다
4. 가져온 문제들을 사용자의 실력에 따라 적절한(학습하기에 가장 적합한)문제를 제공하여 사용자의 경제 문제 풀이 능력을 상승시킨다.

문제 제공(?) : 사용자의 관심 분야, 가입 시에 획득한 개인 프로파일 등을 그래프 형태로 모델링한 후 유사 이력을 지닌 다른 사용자의 패턴을 분석하여(협업 필터링) 효과적으로 문제를 추천해보자.
